* Engine

** TODO Shrink upd-epub-to-jp.tl's packed error message
Failure JSONs now carry the failed run's ~reports~/~log~ across all three
front-ends, so the script can go back to a plain ~(error ...)~ with the
diagnostics living in reports/log again.

** TODO One-shot closing diff: track the primary buffer [#B]
The run's closing ~diff~/~len_*~ compare the *initial primary* buffer against
whatever buffer is *current at exit*. A program invoked with ~--infile~ (so
the primary is an empty ~*stdin*~) that ends on a ~find-file~'d document gets
the whole document rendered as one giant insertion — ulysses-annotated's
~upd-epub-to-jp.tl~ emitted ~2 MB of escaped JSON per run until it switched
back to the primary before exiting (its add-anno / place-diff siblings carry
the same ~set-buffer~-home workaround). Diff the primary buffer's own
before/after instead — or make the closing diff suppressible — and the three
scripts can drop the workaround.

** TODO O(n) buffer-level replace [#B]
~replace-string~ / ~replace-regexp~ on a large Quire are effectively O(n²):
the unbounded windowed search re-materializes point→end per match (a 43 MB
~replace-regexp~ hung minutes). Make them a single streaming rewrite, like
~replace-regexp-in-string~ already is.

** TODO Parallel search/replace [#C]
Needs a resumable streaming DFA (~regex-automata~) for cross-chunk matches.
Related: bounded/streaming search would also fix the conflict builtins'
full-buffer ~scan~ cost on a huge Quire (each unbounded ~search_forward~ miss
materializes point→point-max; every ~conflict-*~ call re-scans, and
~conflict-keep~ scans twice — find + remaining count).
/Exit: global-replace a multi-GB file across cores; ulysses bench beats
legacy ~mime~./

** TODO Reads from a warm Quire can serve corrupted bytes after an external IN-PLACE overwrite
Found by live-firing the MCP server: the warm buffer mmaps the file, so an
external in-place truncate+rewrite mutates the mapped bytes under the piece
tree — ~read_region~ returned the new content padded with NULs, and a
subsequent replace failed with a misleading "no match". Rename-based writers
(editors, git, our own atomic save) are safe — the old inode survives. The
stale guard protects SAVES only; reads stay silent even though
~session_status~ already knows ~stale:true~. Options: surface staleness on
the read path (cheap warning), snapshot at open, or the long-planned
paged-LRU reader replacing mmap (also removes the SIGBUS-on-truncate risk).

** TODO Stale-save escape hatch
When the ~FileStamp~ guard refuses a save (visited file changed externally),
the only outs are re-opening — which discards the warm buffer's edits — or
saving elsewhere. Add ~revert-buffer~ (re-read from disk, Emacs parity) and
decide whether a force-save belongs next to it or stays deliberately absent.

** TODO find-file dedup by path, not basename
Two files sharing a basename collide (the second aliases the first). Key
buffers by visited path and uniquify the name Emacs-style (~doc.txt<2>~);
~unique_buffer_name~ already exists in engine.rs. Same fix for
~find-file-noselect~.

** TODO replace-match FIXEDCASE/LITERAL args
It always expands backrefs; the optional Emacs arguments are unimplemented.

** TODO Line anchors in windowed regex search
~^~ / ~$~ anchor the *windowed slice*, not lines: the search builtins apply
the pattern to ~text[start..end]~ without the multi-line flag, so
~(re-search-forward "^foo")~ misses every line start except the window's own
(Emacs regex ~^~ means line start). Options: compile searches with ~(?m)~ by
default (Emacs parity — check ~$~/~\z~ interactions and the ulysses
patterns), or document the divergence loudly. Surfaced by the review of the
conflict overview's occur suggestion, which had to drop its anchor.

** TODO forward-line shortfall at the top boundary
Landing on a line beginning that sits exactly at point-min (or buffer start)
reports a shortfall of 1 where Emacs reports 0 — the position is right, only
the return value diverges. Pre-existing, identical in both stores; fixing it
means teaching the backward scan that "no newline above, but already at a
line beginning" is a complete move.

** TODO Per-program resource limits
A wall-clock/CPU bound per program — blocked on tulisp cooperative
cancellation (eval interruption), see the wishlist below.

* Structural editing

** TODO Conflict hunk context helper
Deciding a resolution usually needs the code AROUND the hunk; today the agent
parses ~@pos~ out of the overview and calls ~view~. A
~(conflict-context &optional N LINES)~ — the hunk plus a few surrounding
lines, rendered — would make the decision view one call.

** TODO treesit × narrowing
The treesit builtins parse the FULL document and report absolute positions
regardless of narrowing — the one vocabulary family that doesn't compose with
the (now coherent) narrowing semantics: ~treesit-beginning-of-defun~ can move
point outside the restriction. Decide with the persistent-tree work below:
either clamp treesit motion into the accessible region, or document that the
structural layer is deliberately whole-document.

** TODO Persistent per-Session tree (the lever) [#B]
Keep the parse on the ~Session~ instead of re-parsing every call. Unlocks:
nodes as first-class values (a live tree to reference, like markers),
relational navigation + fields (~node-parent~ / ~child~ / ~next-sibling~ /
~child-by-field-name~), and incremental re-parse (~InputEdit~s, not O(buffer)
per call).

** TODO AST-edit ops [#C]
~replace-node~ / ~wrap-node~ / ~raise-node~ / ~kill-node~ / insert-sibling —
thin wrappers over delete+insert once the persistent tree lands.

** TODO More languages [#C]

** TODO Paged-LRU reader replacing mmap
The real fix for the in-place-overwrite hazard: a warm Quire mmaps the
file, so an external in-place truncate+rewrite mutates the mapped bytes
under the piece tree (rename-based writers are safe — the old inode
survives). Reads now WARN when the stamp is stale, but the bytes are
still corrupt; a paged reader (or snapshot-at-open) makes them
impossible, and also removes the SIGBUS-on-truncate risk.

** TODO Span-bounded search (anchors past an explicit BOUND)
With an explicit search BOUND, ~$~/~\b~ treat the haystack cut as a
boundary where Emacs consults the real buffer (it backtracks the match
to fit the bound AND evaluates assertions against the text past it).
The high-level ~regex~ API can't express both; ~regex-automata~'s
~Input::span~ can — matches confined to a span, look-around reading
outside it. Switch the searches to it and the divergence (documented on
~Buffer::re_search_forward~) closes.

** TODO replace-match FIXEDCASE/LITERAL args
It always expands backrefs; the optional Emacs arguments are unimplemented.

** TODO forward-line shortfall at the top boundary
Landing on a line beginning that sits exactly at point-min (or buffer start)
reports a shortfall of 1 where Emacs reports 0 — the position is right, only
the return value diverges. Pre-existing, identical in both stores; fixing it
means teaching the backward scan that "no newline above, but already at a
line beginning" is a complete move.
